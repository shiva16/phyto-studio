### 3. CF Worker AI fix (`worker/index.js` + `deploy-worker.yml`)

**Root cause of "Worker error 405":** The AI Worker URL was saved in the browser as `studio.carnivorousplants-in.workers.dev` (missing `https://`). Without the protocol, `fetch()` resolved it as a relative path against the GitHub Pages origin, and GitHub Pages returns HTTP 405 for all POST requests.

**Fixes applied:**
- `v2.html` `saveDashSettings()`: auto-prepend `https://` if the saved URL has no protocol
- `worker/index.js`: removed `response_format: { type: 'json_object' }` (caused issues with some model versions), added `GROQ_API_KEY` guard (returns clear 500 if secret missing), switched from `.json()` to `.text()` + parse for better error passthrough, restored fast model to `llama-3.1-8b-instant`
- `deploy-worker.yml`: simplified to just `wrangler deploy` — GROQ_API_KEY lives as a secret on the CF Worker directly, no need to push it through GitHub Actions

**Key lesson:** GROQ_API_KEY is stored directly on the Cloudflare Worker (set once via dashboard or `wrangler secret put`) — it does NOT go through GitHub Actions secrets. GitHub Actions only deploys the code.

**Verified working:** `https://studio.carnivorousplants-in.workers.dev/generate` returns correct Groq responses. AI generator confirmed working after adding `https://` to the saved URL.
