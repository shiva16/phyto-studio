### 2. Comprehensive Help Panel (`v2.html`)
Replaced the minimal `openShortcutsPanel()` (shortcuts-only modal) with a full `openHelpPanel(tab)` panel modelled after the tcplants IoT help design.

#### Structure
- Full-screen modal overlay (max 1020px × 90vh)
- Left navigation sidebar: 6 sections with emoji icons, category group labels
- Right scrollable content area: section panels with `.hcard` content cards
- Search bar at top — filters across ALL sections simultaneously using `data-tags` + `textContent`; hides sections with no matches; shows "No results found" state

#### Sections built
| Section | Cards | Key content |
|---|---|---|
| Getting Started | 4 | What is Phyto Studio, GitHub PAT sign-in (step-by-step), AI Worker setup (table of secrets), data storage (enc file table) |
| Dashboard | 6 | Brand tabs & search, creating presentations (step-by-step), status badges (table with color tags), card info panel, settings, preview slides |
| Editor | 10 | Toolbar tools (table), canvas interaction (table), slide panel, properties panel, shapes (all 8), text & formatting (table), arrange & align, saving & undo/redo, speaker notes, zoom & fit, presentation settings |
| AI Content | 4 | Prerequisites, AI generator (step-by-step), AI brief at creation, tips for better prompts |
| Exporting | 3 | PPTX download (with caveats), publish to web/Reveal.js (step-by-step, key fragment explanation), public catalogue |
| Shortcuts | 2 | Full keyboard shortcuts table (all categories), menu bar reference |

#### Wiring
- Dashboard `?` button → `openHelpPanel('start')`
- Editor `?` button → `openHelpPanel('shortcuts')`
- `?` key in editor → `openHelpPanel('shortcuts')`
- Esc closes the help panel
- Click outside modal closes it

#### CSS added
~80 lines of new CSS: `.help-overlay`, `.help-modal`, `.help-nav`, `.help-nav-btn`, `.help-content`, `.help-section`, `.hcard`, `.hcard-title`, `.hcard-body`, `.htag` (5 variants), `.hstep`, `.hkbd-row`, `kbd`, `.hwarn`, `.hinfo`, `.hgrid2`, responsive breakpoint at 700px

#### Design choices
- `.htag` colour-coded system: teal=brand, green=status, amber=warn, red=danger, gray=neutral
- `.hwarn` amber callout box for important caveats
- `.hinfo` teal info box for tips
- `.hstep` numbered circle steps for workflows
- Tables used throughout for reference data
- `data-tags` on every card for keyword search
- JS syntax verified clean (`node -e "new Function()"`)
