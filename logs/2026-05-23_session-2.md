### PhytoTerm — AI-powered TUI embedded in Phyto Studio

#### Architecture
- `#phytoterm` div is a sibling to `#app`, never inside it — survives all navigation (just `display:none` toggle)
- xterm.js v5.3.0 + xterm-addon-fit v0.8.0 for responsive terminal emulation
- Puter.js for AI (Claude Sonnet 4.6, DeepSeek R1, GPT nano) — one-time popup login, persists in localStorage indefinitely
- All data stored in public GitHub repo `shiva16/phyto-term` as `.enc` files (AES-256-GCM, PAT-derived key)
- PAT is the only credential — used for GitHub API and as encryption key source

#### `phytoterm.html` — standalone TUI (new file)

- Dark terminal-themed PAT login screen with encryption notice
- Own `deriveKey`/`aesEncrypt`/`aesDecrypt` (identical algorithm to Studio)
- Auto-login from sessionStorage on return
- All 30+ commands identical to embedded version
- Same `phyto-term` encrypted data store — sessions sync between embedded and standalone
- `ptSignOut()` returns to login screen (vs `closePhytoTerm()` in embedded)

#### Security model
- `phyto-term` repo is PUBLIC — safe because every single file is AES-256-GCM ciphertext
- Key = PBKDF2(PAT, 'phyto-studio-2026', 100k iterations, SHA-256)
- Without the PAT, all repo contents are unreadable random noise
- `phyto-term` stores ONLY: `sessions/YYYY-MM-DD_session-N.enc`, `context/state.enc`, `context/conversation.enc`
- `phytoterm.html` (app code) lives in `phyto-studio` — zero plaintext data in `phyto-term`
