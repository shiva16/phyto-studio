# Session 2 — 2026-05-24

## Goal
Investigate and fix em-dash corruption in `data/catalogue.json` and `data/catalogue.enc`.

## Root cause

**Commit 4b4703** introduced corrupted titles (`â` instead of `—`) in `catalogue.json`.

The bug is an asymmetry in the GitHub API read/write paths:

- **Write** (`ghPut`): `btoa(unescape(encodeURIComponent(content)))` — correctly encodes multi-byte UTF-8
- **Read** (`loadCatalogue`, `updatePublicCatalogue`, `loadPresJSON`): `JSON.parse(atob(...))` — `atob` returns a binary string (one char per raw byte); `JSON.parse` misinterprets UTF-8 multi-byte sequences as individual Latin-1 code points

For `—` (UTF-8: `E2 80 94`):
- Read as: chars at 0xE2 (`â`), 0x80, 0x94 in JS memory
- Written back: each char UTF-8-encoded → 6 bytes `C3 A2 C2 80 C2 94` (double-encoded)

The encrypted path (`ghGetEnc`) was unaffected — it decrypts via `new TextDecoder().decode(pt)` which correctly handles UTF-8.

Entries from `catalogue.json` were being loaded as "samples" in `loadCatalogue` with the buggy `atob` decode, merged into `S.presentations`, then saved to `catalogue.enc` with the corruption — propagating the bug into the encrypted store.

## Fixes

### 1. `data/catalogue.json` — data fix (commit fd7fef9)
Binary-replaced `C3 A2 C2 80 C2 94` → `E2 80 94` in the three corrupted titles:
- `phytolabs-services-2026`
- `tcplants-catalogue-2026`
- `carnivorous-plants-catalogue-2026`

### 2. `v2.html` — code fix (commit c9861b9)
Changed all plain-JSON reads from `JSON.parse(atob(...))` to `JSON.parse(decodeURIComponent(escape(atob(...))))` — the inverse of the write encoding — in three places:
- `loadCatalogue` line 640 — reads `catalogue.json` samples on startup
- `updatePublicCatalogue` line 667 — reads existing `catalogue.json` on every save
- `loadPresJSON` line 679 — reads legacy plain `.json` presentation fallback

### 3. `data/catalogue.enc` — data fix (via Node.js script, not committed)
Used same PBKDF2 + AES-256-GCM key derivation as the Studio (`phyto-studio-2026` salt, 100k iterations) with PAT `<redacted>` to decrypt, patch the three titles, re-encrypt, and push directly to GitHub API.

Verified decryption: all 15 entries read back with correct `—` in all titles.
