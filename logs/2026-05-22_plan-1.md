# Plan — Phyto Studio — 2026-05-22 Session 1

## Goal
Build `studio.phytolabs.in` — a multi-brand AI-powered marketing content studio for Phyto Evolution Pvt. Ltd.  
Serves all 3 brands: tcplants.in · carnivorousplants.in · phytolabs.in

## Stack
- **Frontend**: Single-file vanilla HTML/CSS/JS → GitHub Pages (`shiva16/phyto-studio`)
- **AI proxy**: Cloudflare Worker (`phyto-studio`) — Groq key stored as CF secret
- **Storage**: GitHub API → files in this repo
- **Domain**: `studio.phytolabs.in` → CNAME → `shiva16.github.io`

## Architecture decisions
- Each presentation has 3 artifacts:
  1. `data/presentations/[slug].json` — editable slide data
  2. `view/[slug]/index.html` — published Reveal.js HTML (view-only)
  3. Entry in `data/catalogue.json` — metadata for archive listing
- AI: CF Worker proxies Groq; browser never sees the API key
- Auth: GitHub PAT entered at session start, stored in sessionStorage only
- Worker URL: configurable in studio settings (workers.dev URL after CF deploy)
- Exports: Publish online (Reveal.js) + PPTX download (PptxGenJS) + PDF (print mode)

## Presentation lifecycle
draft → published → archived (manual)
Expiry date (optional): auto-hides from public archive when today > expiry
